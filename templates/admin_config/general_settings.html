Archiving

When archiving is enabled, deleting a test plan preserves it in an archive for a configurable number of days. Archived test plans can be restored or permanently deleted from the Archived Test Plans page. Run history is preserved while archived.

When enabled, deleted test plans are archived instead of permanently removed. When disabled, test plans are permanently deleted along with their associated run history.
Number of days to keep archived test plans before automatic cleanup. Default: 30.
How It Works
  • Archiving ON: Deleted test plans are moved to an archive table with their file contents preserved. Run history is kept intact for potential restoration.
  • Archiving OFF: Test plans are permanently deleted along with all associated runs, screenshots, baselines, and AI analyses.
  • Auto-cleanup: Expired archives are automatically removed when their retention period ends.
  • Restore: Archived test plans can be restored from the Archived Test Plans page.

About Test Plan Types

Test plan types categorize your test plans in the Test Plans list and Builder. The value is stored in the database and the label is what users see.

These are purely for organization — no type triggers special behavior. Baseline comparison runs automatically for any test plan that has stored baselines, regardless of type.

About OIDC

SCOUT supports federated login via OpenID Connect (OIDC). Any standards-compliant identity provider can be used — Authentik, Okta, Keycloak, Auth0, Google Workspace, etc.

Required IdP claims:

  • cap:site_scout = true — grants access
  • cap:admin_scout = true — grants admin
  • email — for user matching

Users are auto-provisioned on first login. Admin status syncs from the IdP on every login.
