Archiving

When archiving is enabled, deleting a test script preserves it in an archive for a configurable number of days. Archived tests can be restored or permanently deleted from the Test Archives page. Run history is preserved while archived.

When enabled, deleted tests are archived instead of permanently removed. When disabled, tests are permanently deleted along with their associated run history.
Number of days to keep archived tests before automatic cleanup. Default: 30.
How It Works
  • Archiving ON: Deleted tests are moved to an archive table with their file contents preserved. Run history is kept intact for potential restoration.
  • Archiving OFF: Tests are permanently deleted along with all associated runs, screenshots, baselines, and AI analyses.
  • Auto-cleanup: Expired archives are automatically removed when their retention period ends.
  • Restore: Archived tests can be restored from the Test Archives page.

About Script Types

Script types categorize your test scripts in the Scripts list and Builder. The value is stored in the database and the label is what users see.

These are purely for organization — no script type triggers special behavior. Baseline comparison runs automatically for any script that has stored baselines, regardless of type.
