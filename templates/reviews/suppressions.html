About Auto-Dismiss Rules

Auto-dismiss rules prevent known false positives from generating new pending review items on future test runs.

How rules are created:

  • When you dismiss a review issue, SCOUT automatically creates a rule matching that specific screenshot or AI analysis + test plan + environment combination.
  • On future runs, if the same combination is detected, the review is automatically created with "dismissed" status instead of "pending".

Two types of rules:

  • Screenshot — matches by screenshot name + test plan + environment. Prevents flagged screenshot diffs from re-appearing.
  • AI Analysis — matches by analysis type + item + test plan + environment. Prevents AI text/visual issues from re-appearing for the same item.

Removing a rule: Delete the rule here and the issue will be flagged again on the next run.
